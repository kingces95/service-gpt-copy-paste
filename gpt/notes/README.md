# Notes

This directory records architectural notes: moments where the design changes
direction, a naming problem becomes clear, or a future quest gets its origin
story.

Unlike commit notes, notes do not have to correspond to a single commit.
They capture why a thread started so later refactors can remember the original
pressure instead of only seeing the final shape.

Prefer filenames of the form:

```txt
YYYY-MM-DD-short-topic.md
```
