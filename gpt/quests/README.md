# Quests

This directory tracks larger design quests that have emerged in conversation
but should not be started as casual side quests.

Use these notes for work that is bigger than a local cleanup and likely to
change public concepts, naming, tests, or documentation. A quest note should
capture the motivation, lineage, likely direction, and why it is being deferred.

Quest notes are assistant-facing working memory. They are allowed to be rougher
than product docs, but they should be precise enough that a future session can
resume the design without re-deriving it from chat history.
