## Key Direction

The project is a metadata-first, CLI-centered workflow system for AI-operated
automation. It should allow tools to be installed into a sandbox, discovered
with `--help`, composed in a visible REPL loop, and promoted into repeatable
workflows with human-readable state and history.

## New Packages Added

- `@kingjs/function-contract`
- `@kingjs/cursor-checks`

These introduce a small standalone function precondition system:

```js
fn[Preconditions] = [
  null,
  null,
  [
    DefaultConstructible,
    PushBackContainer,
  ],
]
```

The declaration shape is intentionally simple and reflectable.

## Design Ideas Discussed

- XSD-style `[Restrictions]` could fit naturally on `Check` classes.
- Ordinary types in function preconditions can use existing `instanceof`
  semantics.
- Special check types provide custom validation and better error messages.
- Defaults for wrapped functions cannot be reflected reliably from JS default
  parameters, so defaults should be explicit metadata if preconditions need
  them.
