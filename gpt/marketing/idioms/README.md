# Idioms Made Declarative

This directory collects examples where a normal JavaScript idiom is lifted into
metadata that a loader can understand.

Each note should show:

- the raw JavaScript idiom
- the declarative project style
- why the declarative version is better for reflection, docs, validation, or
  composition

When the prose says "any type implementing this concept", "a concrete type can
compose this", or anything similar, continue the example to a logical
conclusion. Show a small type consuming the declaration and one call site that
demonstrates the payoff.
